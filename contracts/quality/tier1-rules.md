# Tier 1 Rules — Definition

Tier 1 means: the entity is structurally complete and stable enough to be used for
certification progression and AI consumption.

Tier 1 rules are evaluated across **all known system versions** of the entity.

## Tier 1 Requirements (Global)
1. Identity keys present and stable (no ambiguous duplicates)
2. Required attributes present per contract
3. Attribute formats valid
4. No unresolved conflicts between designated authorities and observed values
5. Lineage declared where enrichment exists
6. Produces a deterministic “best known record” view + issue list

## Outputs
- Tier assignment (1/2/3)
- Issue list (standard codes)
- Candidate “complete record” projection (best known values + provenance)
